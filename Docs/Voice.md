# Praccy voice

Praccy talks like a calm, quietly confident older sibling. Warm, precise,
never hyped, never scolding. Children read these strings; teachers skim
them too.

## Rules

1. **No exclamation marks.** Anywhere. Not in UI strings, not in copy, not in
   push notifications. A full stop is always enough.
2. **No "Oops!", no "Uh-oh", no "Oh no".** If something fails, say what
   happened in plain language and what the user can do next.
3. **Headlines: ≤ 3 words.** `"Let's play."`, `"Keep going."`, `"All done."`,
   `"The big picture."` are the bar.
4. **Buttons: ≤ 2 words.** `"Mark done"`, `"Start recording"`, `"Re-record"`,
   `"Add todo"`.
5. **No streak-shaming.** Don't tell a student they broke a streak, don't
   guilt them for missing a day. Praccy is a practice companion, not a
   taskmaster.
6. **No pushy push notifications.** If we add push later, the same rules
   apply: factual, short, never demanding.
7. **Teachers are named.** When a task is teacher-assigned, surface the
   teacher's first name ("From Claire") rather than an impersonal "your
   teacher".
8. **No emojis in UI strings.** The mascot and chunky shapes carry the
   warmth; emojis flatten the visual language.

## Tone check, before shipping copy

- Would a 9-year-old understand every word?
- Could a teacher send this to a student without cringing?
- Is it free of hype and guilt?

If any answer is no, rewrite.
